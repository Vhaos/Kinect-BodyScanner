Requirements

The Client

For the purposes of this project, our client was Prof. Phillip Treleaven, a professor of computer science at UCL. Professor Treleaven first led the effort of building and encouraging use of body scanners for the clothing industry. He, alongside with Dr.Tony Ruto, developed the initial sizing software, which scanned and extracted measurements from an already existing point cloud using the Kinect 1.0. Our Secondary point of contact was Dr. Tony Ruto, who we would refer to in the event of any particular technical difficulties as well as in seeking guidance on the quality of our extracted point cloud (Kinect model data).

Project Scope

Our main goal for the project, as was discussed and agreed upon with our client, was to look through the SDK and API of the new Kinect 2.0 to extract a suitable point cloud (Kinect model data). Once extraced, and approved for quality by Dr. Tony Ruto, we could later combine this model with the existing size and measurements extraction software which we receive from Dr. Ruto to create a full, low cost Body Scanner software service; compatible with the Kinect 2.0/

Project Requirements

After our intial meetings with Prof. Treleaven, as well as Dr. Ruto the quality of the point cloud, we concluded on final set of clear requirements that the system must have.

  • Kinect Capture

    - The software firstly needed to be able to capture depth scan from the Kinect 2.0 in order to generate a 3D point cloud of the subject in a simple, quick and easy way - One click run

  • Accuracy of the pointcloud

    - the 3D pointcloud needs to be accurate enough to correctly detect body shape and depth. This meant trying to map out as many points as possible without too much quality of the cloud lost due to noise.

  • Extractibilty

    - Most importantly, the point cloud genereated by our Kinect 2.0 needed to be suitable enough to be extractable for measurements from the existing size and measurement scanner. This required the data to be sent to the size extraction software in a particular format.

  • Display of calculated measurements

    - After data has been extracted the software must also be able to show the calculated measurements as well as the captured model of the subject. An optional feature would be additional model visualisation tools e.g. scaling different parts of the body in size to see the effect.
